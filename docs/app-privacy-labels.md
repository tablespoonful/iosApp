# App Store「App のプライバシー」ラベル対応表

App Store Connect の **App のプライバシー（App Privacy）** 設定は、本サイトのプライバシーポリシー
（`*.html`）と**内容が一致**している必要があります。以下は各アプリの実コードから導いた申告内容です。

> 前提：
> - 全アプリが **Google AdMob**（+ UMP 同意）を組み込み → 広告 SDK 由来のデータ収集を申告。
> - 「収集（Collect）」= 端末外へ送信されること。**端末内で完結する処理は申告不要**。
> - AdMob の申告は Google 公式ガイド [Prepare for Apple's App Store data disclosure requirements](https://support.google.com/admob/answer/9997589) の**標準構成**を前提。追加シグナル（精密位置を広告へ渡す等）は未使用。

## 全アプリ共通（AdMob 由来）
| データ種別 | 具体項目 | 目的 | ユーザーに紐付け | トラッキング利用 |
|---|---|---|---|---|
| 識別子 | デバイスID（IDFA） | サードパーティ広告 / 分析 | いいえ | **はい**（ATT 要求）|
| 使用状況データ | 製品操作・広告データ | サードパーティ広告 / 分析 | いいえ | はい |
| 診断 | クラッシュ・パフォーマンスデータ | アプリの機能 / 分析 | いいえ | いいえ |

- ATT：AdMob + UMP により**トラッキング許可のリクエスト**が出るため、「トラッキングに使用」を **はい**。
- 「ユーザーに紐付け」：ログイン等でユーザーIDと結び付けていないため **いいえ**。

## アプリ別の追加項目
| アプリ | App Store ID | 追加で申告すべきデータ | 根拠（実コード）|
|---|---|---|---|
| 西暦和暦変換器 | 6711329947 | **なし**（共通のみ）| ローカル通知は端末内でスケジュール＝収集なし |
| 10秒止めゲーム | 6733242058 | **なし**（共通のみ）| 記録は端末内保存＝収集なし |
| 外食サーチ | 6736398611 | **位置情報 → 精密な位置**（目的：アプリの機能、紐付け：いいえ、トラッキング：いいえ）| `CLLocationManager` + `MKLocalSearch`/`CLGeocoder` で現在地を Apple のマップサービスへ送信 |
| シンプル写真帳 | 6739175712 | **なし**（共通のみ）| 写真は PhotoPicker で取り込み端末内保存・PDF も端末内生成＝**端末外へ送信しないため写真は「収集」に該当せず** |
| ScheduleManager | 未登録 | **連絡先情報**（氏名・メールアドレス）、**ユーザーコンテンツ**（予定・依頼・その他入力内容）、**識別子**（ユーザーID・デバイスID）、**診断**（クラッシュデータ） | Googleの認証・クラウド保存・通知・クラッシュ解析サービスを利用。目的はアプリの機能および不具合調査。ユーザーに紐付けあり、トラッキング利用なし |

### 注意点
- **外食サーチの位置情報**：目的は「アプリの機能」。広告（AdMob）へ位置を渡していないので「サードパーティ広告」目的では申告しません（コード上も広告へ位置連携なし）。
- **シンプル写真帳の写真**：端末外送信がないため「収集する写真」には該当しません。ただし将来クラウド同期や共有アップロードを追加したら、**写真（User Content）を「収集」に切り替える**必要があります。
- 実装を変更（クラウド化・アカウント連携・解析イベント追加等）したら、本表と `apps.json` の `flags`、および ASC の App Privacy を**必ず同時に更新**してください。

## 反映手順（ASC）
1. App Store Connect → 対象アプリ → **App のプライバシー**。
2. 上表のデータ種別・目的・紐付け・トラッキングを設定。
3. 本サイトのプライバシーポリシー（`docs/../<id>.html`）と齟齬がないか確認。
4. `docs/appstore-urls.md` の URL 設定とあわせて実施。
